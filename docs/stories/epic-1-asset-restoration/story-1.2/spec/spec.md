## 1. Overview

Porting of critical infrastructure scripts for automation, validation, and synchronization from the official repository to the local brownfield environment.

### 1.1 Goals

- Restore scripts for `sync`, `validate`, `bootstrap`, and `agent-orchestration`.
- Fix absolute paths in official scripts to work with project-relative paths.

### 1.2 Non-Goals

- Creating new scripts not present in the official repository.
- Modifying script logic beyond path normalization.

## 3. Technical Approach

### 3.1 Architecture Overview

The @dev agent will perform "Script Sanitization":
1. Copy script from official repository.
2. Replace hardcoded absolute paths with `${PROJECT_ROOT}` or relative paths using `node path.join()` for cross-platform safety.
3. Add `chmod +x` logic if on Unix; provide `.ps1` equivalents for Windows where applicable.
