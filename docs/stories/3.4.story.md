# Story 3.4: Agent Metrics & Observability

## 📖 Story Statement

**As a** System Administrator / Developer
**I want to** collect and expose real-time metrics from all AI agents in the system
**So that** the dashboard can display live status, performance metrics, and error information for each agent.

---

## 🎯 Acceptance Criteria

- [ ] **AC 3.4.1:** Agent metrics middleware/service captures execution data for all agent invocations
- [ ] **AC 3.4.2:** Metrics stored in `agent_metrics` table with < 1s latency
- [ ] **AC 3.4.3:** Dashboard displays all agents with current status (running, idle, error, paused)
- [ ] **AC 3.4.4:** Historical metrics available (last 24h, 7d, 30d views)
- [ ] **AC 3.4.5:** Alerts configured for agent failures/errors
- [ ] **AC 3.4.6:** Agent metrics include: latency, success rate, error count, processed items, resource usage
- [ ] **AC 3.4.7:** No performance impact on agent execution (< 5ms overhead per agent run)
- [ ] **AC 3.4.8:** Metrics retention policy applied (30 days default)

---

## 📋 Description

Implement comprehensive metrics collection for all AIOX agents.

### Metrics Collected Per Agent

- **Performance:**
  - Execution latency (ms)
  - Success/failure rate
  - Throughput (items processed per second)
  - Execution time distribution (p50, p95, p99)

- **Errors:**
  - Error count
  - Error type categories
  - Stack traces (for debugging)
  - Last error timestamp

- **Resources:**
  - Memory usage
  - CPU usage
  - Database query count
  - External API calls

- **Status:**
  - Current state (running, idle, error, paused)
  - Uptime
  - Last execution time
  - Queue depth (if applicable)

### Implementation Approach

1. **Middleware Layer**
   - Wrap agent execution in metric collection
   - Capture start time, end time, status
   - Track resource usage (process.memoryUsage(), etc)
   - No impact on agent logic

2. **Database Writing**
   - Batch writes every 5s (not per-execution)
   - Async write to avoid blocking
   - Fallback to queue if DB unavailable

3. **Real-time Publishing**
   - Publish to Supabase Realtime
   - Dashboard subscribes to updates
   - < 100ms delivery latency
