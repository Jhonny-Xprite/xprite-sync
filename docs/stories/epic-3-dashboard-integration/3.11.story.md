### Primary (Must Have) — Stories

- [ ] Dashboard StoryWorkspace displays stories from project (not FALLBACK_STORIES)

  - Option A: Backend endpoint `GET /api/stories` that reads `docs/stories/` directory
  - Option B: Direct filesystem read from `docs/stories/epic-*/[0-9]*.story.md`
- [ ] Each story displays:
  - Story title
  - Current status (backlog, in-progress, review, done)

  - Story points/effort
  - Assigned executor (if applicable)
  - Progress percentage (% tasks completed)
- [ ] Kanban board columns reflect actual story status
- [ ] Stories grouped by epic with count per status
- [ ] Search/filter functionality works on real data

### Primary (Must Have) — Squads

- [ ] SquadsView displays real squads from API/backend
- [ ] Each squad shows:
  - Squad name
  - Member count
  - Squad type (copywriting, design, creator, orchestrator, etc.)
  - Primary domain/focus
  - Agents assigned to squad
- [ ] Squad list updates when new squads added to project
- [ ] Agent-to-squad mappings are correct

### Secondary (Nice to Have)

- [ ] Epic selector to filter stories by epic
- [ ] Real-time status updates when stories change
- [ ] Quick stats: "12 stories | 3 in-progress | 4 done this week"
- [ ] Squad performance metrics (tasks completed, success rate)
- [ ] Bulk operations: move multiple stories to status
