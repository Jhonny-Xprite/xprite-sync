## 1. Overview

Esta especificação define o conjunto de documentação necessária para o AIOX Dashboard. O objetivo é fornecer guias claros para instalação, operação e desenvolvimento, garantindo que o conhecimento técnico seja compartilhado e acessível a todos os membros da equipe.

### 1.1 Goals

- Atualizar o README.md com visão geral e diagramas. (FR-1, AC-3.9.1)
- Criar guias de configuração e instalação. (FR-2, AC-3.9.2)
- Produzir manual do usuário com walkthrough das funcionalidades. (FR-3, AC-3.9.3)
- Documentar referência de API e guia do desenvolvedor. (FR-4, AC-3.9.4, AC-3.9.6)
- Fornecer guia de troubleshooting para problemas comuns. (FR-5, AC-3.9.5)

### 1.2 Non-Goals

- Criar documentação para o core do AIOX (fora do escopo do Dashboard).
- Implementar um site de documentação vivo (ex: Docusaurus) nesta fase - usaremos arquivos Markdown no repositório.

## 3. Technical Approach

### 3.1 Documentation Strategy

Toda a documentação será escrita em **Markdown** e armazenada na pasta `docs/` e na raiz do pacote `packages/dashboard/`.
- **Diagramas**: Serão criados usando **Mermaid.js** diretamente no Markdown para facilitar a manutenção.
- **Screenshots**: Serão armazenados em uma subpasta `docs/assets/dashboard/`.

### 3.2 Content Structure

1. **README.md**: Porta de entrada rápida.
2. **DASHBOARD_SETUP.md**: Focado em DevOps e Engenheiros de Software (Instalação).
3. **DASHBOARD_USER_GUIDE.md**: Focado em usuários finais e admin (Operação).
4. **DASHBOARD_API.md**: Focado em integradores (Endpoints e Webhooks).
