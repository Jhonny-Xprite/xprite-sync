# Story 3.2: Integração de Dados em Tempo Real

## 📖 Story Statement

**As a** DevOps/Backend Developer
**I want to** establish real-time data flow from backend to dashboard via WebSocket/Realtime protocol
**So that** dashboard users see live updates of agent metrics, system status, and workflow progress without manual refreshes.

---

## 🎯 Acceptance Criteria

- [ ] **AC 3.2.1:** WebSocket or Supabase Realtime connection established and stable
- [ ] **AC 3.2.2:** Real-time updates from backend to dashboard flow with < 100ms latency (p95)
- [ ] **AC 3.2.3:** Automatic reconnection works when connection drops (exponential backoff)
- [ ] **AC 3.2.4:** No memory leaks in subscription/cleanup logic (verified with profiling)
- [ ] **AC 3.2.5:** Message format is versioned and documented
- [ ] **AC 3.2.6:** Error handling for network failures, timeouts, invalid messages
- [ ] **AC 3.2.7:** Dashboard correctly subscribes/unsubscribes to data streams
- [ ] **AC 3.2.8:** All data passed through real-time matches database truth source

---

## 📋 Description

This story establishes the real-time data pipeline that enables the dashboard to receive live updates:

### Real-time Architecture

**Option A: Supabase Realtime (Recommended)**
- Leverage existing Supabase connection
- Use Supabase Realtime for database changes
- Tables subscribe to specific data (agent_metrics, workflows, etc.)
- Automatic filtering via RLS policies

**Option B: WebSocket + Server Sent Events**
- Custom WebSocket server (Socket.io, ws library)
- Backend broadcasts metric updates
- Clients subscribe to channels
- Manual message routing and filtering

### Implementation Tasks

1. **Backend Setup (Node.js)**
   - Install Supabase client: `npm install @supabase/supabase-js`
   - Create real-time service layer
   - Define message schemas (TypeScript interfaces)
   - Implement change detection for agent metrics

2. **Message Format**
   ```typescript
   // Example: Agent Metrics Update
   {
     type: "agent_metric_update",
     version: "1.0",
     timestamp: "2026-03-13T10:30:00Z",
     data: {
       agentId: "agent-001",
       status: "running",
       latency_ms: 45,
       success_rate: 0.98,
       error_count: 2
     }
   }
   ```

3. **Dashboard Frontend (React/Vue/Angular)**
   - Install Supabase JS client
   - Create hook/composable for real-time subscriptions
   - Handle connection state (connecting, connected, disconnected)
   - Update UI reactively as data arrives
   - Implement exponential backoff for reconnection

4. **Connection Management**
   - Health check ping/pong every 30s
   - Automatic reconnect on connection loss
   - Max reconnect attempts with fallback to polling
   - Graceful degradation if real-time unavailable

## 🔧 Technical Details

### Technology Stack
- **Real-time Protocol:** Supabase Realtime (PostgreSQL-based) or WebSocket
- **Backend:** Node.js + Express/Fastify
- **Frontend:** React hooks / Vue composables / Angular services
- **Database:** Supabase PostgreSQL

### Data Flow
```
Backend Process → Insert/Update DB → Supabase Realtime → Dashboard Subscription → UI Update
     (Agent metrics collected)        (Change detection)   (Real-time trigger)  (Display updated)
```

### Performance Requirements
- **Latency:** < 100ms from backend to dashboard (p95)
- **Throughput:** Support 1000+ metrics/sec
- **Reliability:** 99.5% uptime for connection
- **Memory:** No growth over time (proper cleanup)

### Security Considerations
- Use RLS policies to ensure users only see their data
- Authenticate WebSocket connections via JWT
- Validate all incoming messages
- Implement rate limiting per user/connection

### Supabase Realtime Setup
```javascript
// Backend: Listen for changes
const subscription = supabase
  .from('agent_metrics')
  .on('*', payload => {
    console.log('Change received!', payload)
    broadcast(payload) // Send to connected clients
  })
  .subscribe()

// Frontend: Subscribe to updates
const subscription = supabase
  .from('agent_metrics')
  .on('*', (payload) => {
    updateMetrics(payload.new)
  })
  .subscribe()
```
