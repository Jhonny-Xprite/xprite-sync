# Story 3.3: Schema & RLS Policies para Dashboard

## 📖 Story Statement

**As a** Database Engineer / Backend Developer
**I want to** design and implement database schema for dashboard data with Row Level Security (RLS) policies
**So that** dashboard can securely store and query agent metrics, system metrics, workflows, and logs with proper data isolation per user/team.

---

- [ ] **AC 3.3.1:** Database schema created with tables: `agent_metrics`, `system_metrics`, `workflow_logs`, `user_activity`
- [ ] **AC 3.3.2:** All tables have proper indexes for common queries (performance < 200ms)
- [ ] **AC 3.3.3:** RLS policies implemented and tested - users see only their data
- [ ] **AC 3.3.4:** RLS policies prevent unauthorized data access (security tested)
- [ ] **AC 3.3.5:** Migrations created and safely reversible (rollback tested)

## 📋 Description

This story establishes the database foundation for dashboard data storage with security enforcement through RLS.

### Schema Design

**Table 1: agent_metrics**
```sql
CREATE TABLE agent_metrics (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  agent_id VARCHAR(255) NOT NULL,
  team_id UUID NOT NULL REFERENCES teams(id),
  status VARCHAR(50), -- running, idle, error, paused
  latency_ms INTEGER,
  success_rate NUMERIC(4,2),
  error_count INTEGER,
  processed_count INTEGER,
  memory_usage_mb INTEGER,
  cpu_percentage NUMERIC(5,2),
  recorded_at TIMESTAMP DEFAULT now(),
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

CREATE INDEX idx_agent_metrics_team_id ON agent_metrics(team_id);
CREATE INDEX idx_agent_metrics_agent_id ON agent_metrics(agent_id);
CREATE INDEX idx_agent_metrics_recorded_at ON agent_metrics(recorded_at DESC);
```

**Table 2: system_metrics**
```sql
CREATE TABLE system_metrics (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  team_id UUID NOT NULL REFERENCES teams(id),
  cpu_percentage NUMERIC(5,2),
  memory_percentage NUMERIC(5,2),
  memory_used_gb NUMERIC(10,2),
  memory_total_gb NUMERIC(10,2),
  disk_used_gb NUMERIC(10,2),
  disk_total_gb NUMERIC(10,2),
  network_in_mbps NUMERIC(10,2),
  network_out_mbps NUMERIC(10,2),
  db_connections INTEGER,
  db_query_time_ms NUMERIC(10,2),
  api_requests_per_sec INTEGER,
  api_error_rate NUMERIC(4,2),
  recorded_at TIMESTAMP DEFAULT now(),
  created_at TIMESTAMP DEFAULT now()
);

CREATE INDEX idx_system_metrics_team_id ON system_metrics(team_id);
CREATE INDEX idx_system_metrics_recorded_at ON system_metrics(recorded_at DESC);
```

**Table 3: workflow_logs**
```sql
CREATE TABLE workflow_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workflow_id VARCHAR(255) NOT NULL,
  team_id UUID NOT NULL REFERENCES teams(id),
  status VARCHAR(50), -- pending, running, completed, failed
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  duration_ms INTEGER,
  error_message TEXT,
  metadata JSONB,
  created_at TIMESTAMP DEFAULT now()
);

CREATE INDEX idx_workflow_logs_team_id ON workflow_logs(team_id);
CREATE INDEX idx_workflow_logs_workflow_id ON workflow_logs(workflow_id);
CREATE INDEX idx_workflow_logs_started_at ON workflow_logs(started_at DESC);
```

**Table 4: user_activity**
```sql
CREATE TABLE user_activity (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id),
  team_id UUID NOT NULL REFERENCES teams(id),
  action VARCHAR(100),
  resource_type VARCHAR(100),
  resource_id VARCHAR(255),
  changes JSONB,
  created_at TIMESTAMP DEFAULT now()
);

CREATE INDEX idx_user_activity_team_id ON user_activity(team_id);
CREATE INDEX idx_user_activity_user_id ON user_activity(user_id);
CREATE INDEX idx_user_activity_created_at ON user_activity(created_at DESC);
```

### RLS Policies

```sql
-- agent_metrics RLS
ALTER TABLE agent_metrics ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view their team's agent metrics"
  ON agent_metrics FOR SELECT
  USING (
    team_id IN (
      SELECT team_id FROM team_members
      WHERE user_id = auth.uid()
    )
  );

CREATE POLICY "Service can insert metrics"
  ON agent_metrics FOR INSERT
  WITH CHECK (true); -- Backend service inserts

-- Similar policies for other tables...
```
